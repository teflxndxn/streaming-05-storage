## Custom Project

### Dataset

The dataset used by the Kafka producer is `sales.csv`.

It contains:

- order ID
- product ID
- region ID
- quantity
- unit price
- payment method
- customer information
- transaction details

I used the original sales dataset provided with the project.

### Kafka Messages

The producer sends sales transaction records through Kafka.

- Topic: `product-sales-case`
- Message key: `region_id`

I modified the producer to display quantity values
before messages are sent.

### Consumer Processing

The consumer receives Kafka sales messages,
validates them, enriches them with derived fields,
and stores them in CSV and DuckDB outputs.

My custom consumer logs:

- quantity values
- running statistics
- accepted messages

### Experiments

#### Phase 4

I created custom versions of:

- kafka_admin_blessing.py
- kafka_producer_blessing.py

I added custom logging and quantity tracking.

#### Phase 5

I created:

### Results

The producer successfully sent messages to Kafka.

The consumer successfully processed the messages,
stored the results, tracked quantities,
and applied discounts to qualifying orders.

### Interpretation

This project demonstrated how Kafka streams data
from a producer to a consumer in real time.

Compared to the original example,
I added quantity tracking, discount calculations,
and custom validation rules.

The streaming workflow showed how businesses can
process sales transactions, apply business rules,
and generate information for analysis and
decision-making.
